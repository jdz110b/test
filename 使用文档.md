# 语料评测平台 - 使用文档

## 1. 平台简介

语料评测平台是一款基于大语言模型的语料质量评测工具，支持对 Question 语料和 QA 对语料进行多维度分类、评估和打分。平台通过调用兼容 OpenAI Chat Completion API 的大模型服务（如阿里云通义千问），实现自动化的语料评测流程。

### 核心能力

| 功能 | 说明 |
|------|------|
| 主观/客观分类 | 判断问题是主观题还是客观题，客观题自动生成标准答案 |
| 难度等级分类 | 将问题按 L1(简单)、L2(中等)、L3(困难) 三级划分 |
| 语料质量评估 | 从清晰度、完整性、规范性、价值性四维度综合评分(1-10) |
| 领域/场景分类 | 将问题归类到技术、科学、教育、医疗、法律等领域 |
| 意图识别 | 识别用户意图类型（内部/外部信息查询、任务执行、创意生成等） |
| 自定义分类 | 用户自行定义分类类别，由大模型进行归类 |
| Answer 打分 | 对 QA 对中的答案从准确性、完整性、清晰度、相关性进行评分 |
| 结果导出 | 导出评测结果为格式化的 Excel 文件 |

## 2. 环境要求与安装

### 2.1 环境要求

- Python 3.10+
- 可访问的 OpenAI 兼容 API 服务（如阿里云 DashScope、OpenAI 等）

### 2.2 安装步骤

```bash
# 克隆或拷贝项目到目标目录
cd /path/to/llmCreator

# 创建虚拟环境（推荐）
python3 -m venv venv
source venv/bin/activate  # Linux/Mac
# venv\Scripts\activate   # Windows

# 安装依赖
pip install -r requirements.txt
```

依赖列表：
- `flask` - Web 框架
- `flask-sqlalchemy` - ORM 数据库操作
- `requests` - HTTP 请求
- `openpyxl` - Excel 文件生成
- `chardet` - 文件编码自动检测
- `gunicorn` - 生产级 WSGI 服务器（可选，Linux/Mac）

### 2.3 启动服务

**方式一：使用启动脚本（推荐）**

```bash
chmod +x start.sh stop.sh
./start.sh          # 启动服务
./stop.sh           # 停止服务
```

可通过环境变量自定义配置：

```bash
HOST=0.0.0.0 PORT=8080 WORKERS=8 ./start.sh
```

**方式二：直接运行**

```bash
python3 app.py
```

启动成功后，访问 `http://localhost:5000` 即可打开平台。

## 3. 配置大模型

使用平台前必须先配置大模型 API，进入页面顶部导航栏的 **配置管理** 页面。

### 3.1 快速填充模板

页面提供了阿里云通义千问系列模型的快速模板：

| 模板 | 模型 | API 地址 |
|------|------|----------|
| 通义千问 Qwen-Max | qwen-max | https://dashscope.aliyuncs.com/compatible-mode/v1 |
| 通义千问 Qwen-Plus | qwen-plus | https://dashscope.aliyuncs.com/compatible-mode/v1 |
| 通义千问 Qwen-Turbo | qwen-turbo | https://dashscope.aliyuncs.com/compatible-mode/v1 |
| 通义千问 Qwen-Long | qwen-long | https://dashscope.aliyuncs.com/compatible-mode/v1 |

选择模板后，配置名称、API URL、模型名称会自动填入，只需补充 **API Key** 即可保存。

### 3.2 手动配置

填写以下字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| 配置名称 | 自定义名称，便于识别 | `通义千问 Qwen-Max` |
| API URL | 兼容 OpenAI Chat Completion API 的地址 | `https://dashscope.aliyuncs.com/compatible-mode/v1` |
| API Key | API 密钥 | `sk-xxxxxxxxxxxxxxxx` |
| 模型名称 | 模型标识 | `qwen-max` |
| 代理地址（可选） | HTTP 代理地址，需通过代理访问 API 时填写 | `http://proxy.company.com:8080` |
| 验证 SSL 证书 | 默认开启，自签名证书环境可取消勾选 | - |
| 设为默认模型 | 勾选后该配置将作为默认模型使用 | - |

### 3.3 测试连接

保存配置后，在已有配置列表中点击 **测试** 按钮，平台会向 API 发送一条测试请求。

- 显示绿色提示 `连接测试成功` 表示配置正确
- 显示红色错误信息时，请检查 API URL、API Key、网络连通性或代理设置

## 4. 上传语料文件

在首页左侧的 **上传语料文件** 区域操作。

### 4.1 语料类型

| 类型 | 说明 | 适用场景 |
|------|------|----------|
| Question 语料 | 仅包含问题 | 问题分类、质量评估、意图识别 |
| QA 对语料 | 包含问题和答案 | 除以上功能外，还支持 Answer 打分 |

### 4.2 文件格式要求

**CSV 格式：**

- 自动检测文件编码（支持 UTF-8、GBK、GB2312、ANSI 等）
- 自动识别列头，支持的列名映射如下：

| 问题列 | 答案列 |
|--------|--------|
| `question`, `q`, `问题`, `query`, `prompt`, `input` | `answer`, `a`, `答案`, `response`, `output`, `reply` |

- 如果没有匹配的列头，自动使用第一列作为问题，第二列作为答案
- 支持逗号、Tab、分号、竖线作为分隔符

Question 语料 CSV 示例：
```csv
question
什么是机器学习？
Python 如何读取文件？
地球的半径是多少？
```

QA 对语料 CSV 示例：
```csv
question,answer
什么是机器学习？,机器学习是人工智能的一个分支...
Python 如何读取文件？,可以使用 open() 函数...
```

**TXT 格式：**

- Question 语料：每行一个问题
- QA 对语料：每行用 `Tab`、`|||`、`|` 或 `::` 分隔问题和答案

```
什么是机器学习？|||机器学习是人工智能的一个分支...
Python 如何读取文件？|||可以使用 open() 函数...
```

**文件大小限制：** 最大 50MB

### 4.3 上传操作

1. 选择语料类型（Question / QA 对）
2. 点击选择文件
3. 点击 **上传并解析**
4. 上传成功后，文件出现在右侧文件列表中，显示解析到的语料条数

## 5. 语料评测操作

点击文件列表中的文件名或查看按钮，进入 **语料详情** 页面。

### 5.1 选择模型

在操作面板左侧的下拉框中选择要使用的大模型配置。选择会自动记忆，下次打开页面时保持上次选择。

### 5.2 选择评测操作

操作面板中提供以下可勾选的操作：

| 操作 | 说明 | 输出字段 |
|------|------|----------|
| 主观/客观分类 | 判断主观/客观，客观题自动生成答案 | 主观/客观、客观题答案 |
| 难度等级分类 | L1/L2/L3 难度分级 | 难度 |
| 质量评估 | 1-10 分综合质量评估 | 质量分、质量等级 |
| 领域分类 | 归类到预定义领域 | 领域、子领域 |
| 意图识别 | 识别用户意图类型 | 意图、意图(中文)、置信度 |
| 自定义分类 | 用户自定义类别 | 分类 |
| Answer 打分 | 仅 QA 对语料可用 | 评分(1-10)、评分理由 |

**操作说明：**

- 可多选操作，点击 **全选** 快速选中所有操作（自定义分类除外）
- 勾选"自定义分类"后会出现输入框，需填入分类类型（逗号分隔），例如：`数学,物理,化学,生物`
- 多个分类操作会合并为一次 LLM 调用（综合分类），减少 API 调用次数和成本
- Answer 打分始终独立调用

### 5.3 选择 Prompt 模板（可选）

可在下拉框中选择自定义的 Prompt 模板。不选择则使用系统默认 Prompt。

### 5.4 指定评测范围

- 默认对文件中所有语料执行操作
- 可通过勾选语料行前的复选框，只对选中的条目执行操作
- 表头的复选框可以全选/取消全选

### 5.5 执行评测

点击 **执行选中操作** 按钮开始评测：

1. 页面显示进度条
2. 每条语料依次调用大模型进行评测
3. 评测完成后页面自动刷新，结果显示在数据表格中

## 6. 查看与导出结果

### 6.1 查看结果

评测结果直接显示在语料详情页的数据表格中：

- **主观/客观**：蓝色标签"客观"、黄色标签"主观"
- **难度**：绿色"L1(简单)"、黄色"L2(中等)"、红色"L3(困难)"
- **质量**：显示质量分数和等级（high/medium/low）
- **领域**：鼠标悬停可查看子领域
- **意图**：显示中文意图名称，鼠标悬停可查看置信度
- **评分**：绿色(>=7)、黄色(4-6)、红色(<4)

### 6.2 导出 Excel

在语料详情页右上角点击 **导出结果** 按钮，下载包含所有评测结果的 Excel 文件。

导出文件包含以下列：

| 列 | 内容 |
|----|------|
| ID | 序号（按文件内从 1 开始） |
| 问题 | 原始问题文本 |
| 答案 | 原始答案文本（QA 对） |
| 主观/客观 | 客观 / 主观 |
| 难度 | L1(简单) / L2(中等) / L3(困难) |
| 分类 | 自定义分类结果 |
| 客观题答案 | 自动生成的客观题标准答案 |
| 质量评分 | 1-10 综合质量分 |
| 质量等级 | high / medium / low |
| 领域 | 所属领域 |
| 子领域 | 更细分的子领域 |
| 意图 | 英文意图标识 |
| 意图(中文) | 中文意图名称 |
| 意图置信度 | 0-1 之间的置信度 |
| 答案评分 | 1-10 答案评分 |
| 评分理由 | 评分的详细理由 |

## 7. Prompt 模板管理

在 **配置管理** 页面右侧管理 Prompt 模板。

### 7.1 系统默认模板

平台为每种操作预设了默认 Prompt 模板，包括：

- 主观/客观分类
- 难度等级分类
- 自定义分类
- 客观题答案生成
- Answer 打分
- 语料质量评估
- 领域/场景分类
- 意图识别分类
- 综合分类(一次调用)

默认模板不可删除，但可以编辑内容。

### 7.2 自定义模板

点击 **添加模板** 可以创建自定义 Prompt 模板：

1. 填写模板名称
2. 选择模板类型
3. 编写 Prompt 内容，支持以下占位符：
   - `{question}` — 问题文本
   - `{answer}` — 答案文本
   - `{categories}` — 自定义分类列表

4. 保存后，在评测操作时可从下拉框中选择该模板

### 7.3 编辑与删除

- 点击模板名称展开内容
- 修改后点击 **保存修改**
- 自定义模板可以删除，默认模板不可删除

## 8. 意图分类说明

平台的意图识别将问题分为以下 8 种类型：

| 意图 | 中文 | 说明 |
|------|------|------|
| `internal_information_query` | 内部信息查询 | 通用知识、百科知识，大模型自身知识可直接回答 |
| `external_information_query` | 外部信息查询 | 需要外部知识库、实时数据或私有数据才能回答 |
| `task_execution` | 任务执行 | 需要完成具体操作，如写代码、翻译、计算 |
| `opinion_consultation` | 观点咨询 | 希望获得建议、推荐或专业意见 |
| `creative_generation` | 创意生成 | 需要生成创意内容，如写作、起名、设计方案 |
| `chat_conversation` | 闲聊对话 | 日常寒暄、情感交流、无明确目的的对话 |
| `reasoning_analysis` | 推理分析 | 需要逻辑推理、数据分析、对比评估等 |
| `instruction_following` | 指令遵循 | 按特定格式或规则输出，如角色扮演、格式转换 |

**内部 vs 外部信息查询的判断标准：**

- **内部**：常识性知识、公开百科、通用学科知识、广泛已知的历史事件
- **外部**：特定组织/企业内部信息、实时行情/天气/新闻、需查数据库或 API 的问题、私有业务数据

## 10. 常见问题

### Q: 重启服务后配置还在吗？

所有配置（大模型配置、Prompt 模板、语料数据）均持久化存储在 SQLite 数据库中，重启服务不会丢失。平台还内置了自动数据库迁移功能，新增字段会自动添加到现有表中。

### Q: 支持哪些大模型？

支持任何兼容 OpenAI Chat Completion API 格式的大模型服务，包括但不限于：
- 阿里云通义千问（DashScope）
- OpenAI GPT 系列
- Azure OpenAI
- 其他兼容 API 的开源模型部署

### Q: 文件编码问题怎么办？

平台使用 `chardet` 库自动检测文件编码，支持 UTF-8、GBK、GB2312、GB18030、ANSI 等常见编码格式。如果仍出现乱码，建议将文件另存为 UTF-8 编码后重新上传。

### Q: 评测一个文件需要调用几次大模型？

- 选择多个分类操作时，平台会使用 **综合分类** 模式，每条语料只调用一次 LLM 即可完成所有分类维度
- Answer 打分始终需要单独调用一次
- 因此，对于 QA 对语料同时执行所有分类 + 打分，每条语料最多调用 2 次 LLM

### Q: 如何通过代理访问 API？

在大模型配置中填写代理地址即可，格式如 `http://proxy.company.com:8080`。代理字段为可选项，不填则直连。如果使用自签名证书的代理，记得取消勾选"验证 SSL 证书"。

### Q: 如何部署到服务器？

```bash
# 1. 上传项目文件到服务器
# 2. 安装依赖
pip install -r requirements.txt

# 3. 启动服务
./start.sh

# 4. 自定义端口和进程数
HOST=0.0.0.0 PORT=8080 WORKERS=8 ./start.sh

# 5. 停止服务
./stop.sh
```

生产环境建议使用 gunicorn（Linux/Mac）或 waitress（Windows）作为 WSGI 服务器。
